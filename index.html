تقويم هجري كامل لعام 1448هـ يوضح بداية ونهاية كل شهر هجري ومقابله بالتقويم الميلادي والفصل الموسمي المرتبط به. ملاحظة: قد تختلف بداية الأشهر الهجرية يوماً واحداً بحسب مصدر الرصد المعتمد.

توزيع الأسابيع الدراسية لجميع المناطق (ما عدا إدارات التعليم المستثناة)، بمعدل 19 أسبوعاً لكل فصل دراسي.

أنشئ حسابك كمعلم/ة عشان تُعبّى بياناتك (اسمك، اسم مدرستك، اسم مديرك) تلقائياً في أي ملف تصدّره من الموقع، بدون ما تكتبها كل مرة.

تسجيل الدخول / إنشاء حساب

مسجل الدخول:

📋 بياناتي

بعد الحفظ، افتح تبويب "توزيع المنهج" وبتلاقي حقول اسم المدرسة/المعلم/المدير معبّاة تلقائياً من بياناتك.

لوحة تحكم لرفع ملفات توزيع المنهج (Word) وحفظها في قاعدة بيانات سحابية (Firebase)، بحيث تصير التحديثات دائمة ومتاحة لأي شخص يفتح هذي الصفحة، بدون ما نحتاج تعديل الكود يدوياً في كل مرة.

🔒 تسجيل دخول الإدمن

⚠️ لازم تنشئ حساب المستخدم بنفسك أولاً من Firebase Console → Authentication → Add user، وتفعّل طريقة الدخول "Email/Password" من تبويب Sign-in method.

⛔ ما عندك صلاحية إدمن

حسابك مسجّل دخول بنجاح، لكنه غير مضاف كإدمن في قاعدة البيانات. إذا هذا خطأ، تواصل مع مدير النظام.

⚙️ إعداد صلاحيات Firestore (حدّثها الآن بعد إضافة حسابات المعلمين)

روح Firebase Console ← Firestore Database ← تبويب Rules، امسح القديم والصق هذا النص بالضبط ثم اضغط Publish:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /curricula/{document=**} { allow read: if true; allow write: if request.auth != null && exists(/databases/$(database)/documents/admins/$(request.auth.uid)); } match /admins/{uid} { allow read: if request.auth != null && request.auth.uid == uid; allow write: if false; } match /teachers/{uid} { allow read, write: if request.auth != null && request.auth.uid == uid; } } }

بعد نشر القواعد، لازم تضيف نفسك يدوياً كإدمن: من Firestore Database ← تبويب Data ← "Start collection" ← اسم المجموعة admins ← Document ID = الـ UID تبعك (تلقاه من Authentication ← Users بعد ما تسوي حساب) ← أضف أي حقل بأي قيمة (مثلاً role: "admin") واحفظ.
